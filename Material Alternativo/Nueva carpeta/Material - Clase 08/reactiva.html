Programación Reactiva

La programación reactiva es un paradigma de programación basado en la gestión de flujos de datos asíncronos (streams) y en la propagación del cambio.
En front-end, la programación reactiva nos permite modificar la pantalla que el usuario está visualizando en base al valor actual observado.
La programación reactiva está basada en la programación funcional y utiliza los patrones Iterador y Observer.

Patrón Observer

El patrón Observer es muy importante en el concepto de programación reactiva ya que explica perfectamente cómo funciona. Según este patrón, hay un sujeto que es el productor de la información (stream) y por otro lado hay uno o varios consumidores de esta información.
Trabajar con el Patrón Observer implica la creación de un objeto que almacena valores y funciones que reciben callbacks pensadas para que al momento de que cambien los valores almacenados, las funciones ejecuten sus callbacks. De esta forma, las funciones reaccionan a los cambios.

Patrón Iterador

El patrón iterador nos permite está pensado para que podamos almacenar Arrays y recorrerlos sin exponer sus valores a elementos externos para asegurar que, una vez creado, no se realizan modificaciones indeseadas. Trabajar con este patrón, además, reduce significativamente la cantidad de bucles utilizados en nuestro código porque los métodos del iterador están pensados para recorrer el Array sin necesidad de un for. Opcionalmente, es posible agregar un métodos .each/.map/.filter para recorrer el Array contenido con un bucle. Este tipo de métodos se suele agregar cuando deseamos permitir transformaciones al Array original
Un iterador es el objeto que se crea para almacenar el Array y los métodos que se usarán para recorrerlo.

Métodos que deben tener un Iterador:

  • first() -> Retorna siempre el primer elemento del Array. Método obligatorio
  • next() -> Retorna el siguiente elemento en el Array. Este método suele retornar un objeto con las claves value (almacena el valor retornado) y done (almacena un booleano que indica que existe un elemento después del retornado). Este método es obligatorio
  • current() -> Retorna el elemento actual del Array. Método obligatorio.
  • hasNext() -> Retorna un booleano para saber si hay más elementos dentro del Array. Método opcional
  • reset() -> Retorna la iteración al inicio del Array. Método opcional
