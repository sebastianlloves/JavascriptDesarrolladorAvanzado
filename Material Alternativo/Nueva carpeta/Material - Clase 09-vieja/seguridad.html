Seguridad de datos

A la hora de crear los distintos script que utilizaremos en nuestro programas es importante recordar que todo archivo js agregado al documento HTML será accesible desde el Objeto Global Window lo que implica que todas la variables y funciones declaradas dentro de los mismos pueden ser accesibles desde cualquier otro de los archivos declarados en el mismo documento HTML.
Esta es una característica que, bien utilizada, puede resultar bastante útil a la hora de ordenar nuestro código pero puede generar varios problemas relacionados con el espacio de almacenamiento.
Para evitar que el contenido de nuestros script se almacene en el Objeto Global Window podemos ejecutar automáticamente una función anónima contenedora de todo nuestro script, encapsulando de forma efectiva a nuestro código. Las Funciones Anónimas que lo permiten son conocidos como 'Immediately Invoked Function Expression (IIFE)' (Función Anónima Auto-invocada).
Las IIFE también nos sirven para crear scopes únicos dentro de un mismo script lo que nos permitiría repetir nombres de variables y constantes.

Modo Estricto

El modo estricto cambia la sintaxis y el comportamiento en tiempo de ejecución. Los cambios generalmente caen dentro de estas categorías:

  • cambios que convierten equivocaciones en errores (como errores de sintaxis o en tiempo de ejecución)
  • cambios que simplifican cómo se calcula el nombre de una variable particular
  • cambios que simplifican el uso de eval y arguments
  • cambios que hacen más fácil escribir JavaScript "seguro"
  • y cambios que anticipan la evolución futura de ECMAScript
La mayoría de los navegadores ya implementan el modo estricto. Sin embargo, no dependas ciegamente de él, ya que todavía hay numerosas Versiones del navegador utilizadas en la naturaleza que solo tienen soporte parcial para el modo estricto o no lo admiten en absoluto (por ejemplo, Internet Explorer por debajo de la versión 10).
El modo estricto cambia la semántica. Depender de esos cambios provocará equivocaciones y errores en los navegadores que no implementan el modo estricto. Ten cuidado al usar el modo estricto y respalda la dependencia del modo estricto con pruebas de funciones que comprueben si se implementan las partes relevantes del modo estricto. Finalmente, asegúrate de probar tu código en navegadores que admitan y no admitan el modo estricto. Si realizas tus pruebas solo en navegadores que no admiten el modo estricto, es muy probable que tengas problemas en los navegadores que sí lo hacen, y viceversa.
