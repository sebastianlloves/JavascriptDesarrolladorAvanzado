Node.js

Node.js es un entorno en tiempo de ejecución multiplataforma, de código abierto, para la capa del servidor (pero no limitándose a ello) basado en el lenguaje de programación JavaScript, asíncrono, con E/S de datos en una arquitectura orientada a eventos y basado en el motor V8 de Google. Fue creado con el enfoque de ser útil en la creación de programas de red altamente escalables, como por ejemplo, servidores web. Fue creado por Ryan Dahl en 2009 y su evolución está apadrinada por la empresa Joyent.
Node.js adopta para trabajar el Patrón Módulo permitiendo así el encapsulamiento de los distintos componentes de nuestra aplicación lo cual ayuda a la seguridad de la misma, pero hay que tener en cuenta que a la hora de hablar de un módulo en Node.js también podemos estar refiriéndonos a los elementos que propios del entorno de trabajo (equivalente a las API's Web). Para distinguirlos se habla de Módulos Nativos cuando hablamos de elementos propios de Node.js y Módulos cuando hablamos de los distintos archivos js que componen nuestra aplicación.
El trabajo con Node.js nos abre la puerta al mundo del back-end porque, al estar pensado para el trabajo con servidores, podemos configurar nuestros propios servidores y conexiones con bases de datos. Pero esto no es todo, Node.js es también una excelente herramienta para el desarrollo front-end porque nos permite separar nuestros proyectos en dos grandes estructuras conocidas como "Desarrollo" y "Producción"
El código de "Desarrollo" es el código con el que vamos a estar trabajando constantemente, mientras que el código de "Producción" es el resultado final, el trabajo terminado. La gran diferencia entre estas estructuras es que el código de "Producción" se crea como resultado de una serie de procesos aplicados al código de "Desarrollo" pensados para mejorar la performance e integración del mismo.

Inicializando

Al inicializar un proyecto con Node.js vamos a crear un archivo llamado "package.json" que va ser el encargado de almacenar toda la información relativa a las dependencias que utilice nuestro proyecto. Una de las principales ventajas de este archivo es que se puede compartir un proyecto ignorando la carpeta "node_module" (contiene todos los archivos necesarios para cada dependencia y es muy pesada) ya que cualquier persona que reciba nuestro proyecto con su correspondiente "package.json" puede ejecutar el comando npm install para descargar todas las dependencias necesarias

Babel

Babel es un transpilador de código JavaScript. Un transpilador se encarga de leer y adecuar código según los deseado. En el caso de Babel, lo que se busca es convertir el código JS escrito en ECMAScript 2015 o superior a versiones anteriores para mejorar la compatibilidad de todas las funciones con los navegadores y evitar así que nuestras páginas no se ejecuten.
Con Node.js podemos descargar los módulos de Babel para incorporarlo a nuestros proyectos.

Webpack

Webpack es un creador de módulos estáticos para subirlos a la web. Su uso nos permite crear un archivo único, minificado, que contenga toda la lógica de nuestra aplicación mejorando así la velocidad de carga de los archivos en los navegadores.
Webpack trabaja con cuatro conceptos básicos:

  • Entry => archivo desde el que inicia a generar el archivo de Producción
  • Output => archivo de Producción
  • Loaders => paquetes de node que permiten la transformación del código de Desarrollo
  • Plugging => módulos que agregan funcionalidad

Instalación

Para usar Webpack hay que instalar los correspondientes paquetes (dependencias) y para ello necesitamos iniciar un proyecto de node (npm init la carpeta del proyecto)

Configuración

Trabajar con Webpack es relativamente sencillo pero requiere algunas configuraciones. Las siguientes imágenes muestran la configuración básica para el trabajo front-end que debe implementarse en el archivo "webpack.config.js" (debe crearse dentro de la carpeta que tiene el init de node). Se configuran los ambientes de Desarrollo y Producción indicando tipo de transpilación y compilación de archivos e imágenes y se arma un live-server para facilitar el desarrollo.

Dependencias instaladas

  • Transpilar JS (Babel) - babel-loader @babel/core @babel/preset-env
  • Inyectar JS en HTML - html-loader html-webpack-plugin
  • Inyectar CSS en HTML - css-loader mini-css-extract-plugin
  • Manejo de archivos - file-loader
  • Compresión de imágenes - image-webpack-loader
  • Live-server - webpack-dev-server (configurado en package.json => "start": "webpack serve --mode development --open")
